# Agent Memory — LinkedIn Post Writer

## Voice Quick Rules (Non-Negotiable)
- "tú" always, never "usted"
- English tech terms kept in Spanish prose (hooks, output, override, feature, workflow, revenue-based)
- Emoji system: • for lists, → for outcomes, ✔ for validation, ‣ for sub-bullets — punctuation not decoration
- Builder Mode > Philosopher Mode — every post must answer "what did I build/do/learn?"
- Never copy-paste a top-performing post formula with different content — Irina will catch it

## Structures That Work
- Field Report + Pattern Recognition: Hook (authority from experience) → concrete bullet examples → reframe for reader → CTA with specific question
- BAB framework works well for field reports: Builder experience → Accumulated insight → Belief shift for audience
- "The technology is the easy part. The decisions are the hard part." — this structure (short declaration → longer proof) is clean and on-brand

## CTA Style
- Specific question > generic "¿qué opinas?"
- Best CTAs invite readers to share their OWN experience with the pattern described
- Example that worked: "¿Cuándo fue la última vez que el AI te dio algo que tuvieras que corregir porque sonaba bien pero no era tuyo?"

## Bullet Usage
- Use • bullets when listing real examples from experience — keeps the list grounded and scannable
- Don't exceed 6-7 bullets in a list — beyond that, trim ruthlessly
- Each bullet must be a standalone proof point, not a variation of the same idea

## Confirmed Patterns (Post #1 — Field Report)
- Short declarative sentences land better than complex ones for major claims
- Contrast structure works: "El AI ejecuta / Yo decido" — simple, memorable, shareable
- Quotable closing line increases shareability: "Ese siempre fue tuyo" as a revelation ending
- Avoid labeling the post as a "reporte de campo" explicitly — just write it, don't narrate it
