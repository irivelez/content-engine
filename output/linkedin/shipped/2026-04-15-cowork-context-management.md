No se trata de mejores prompts.
Se trata de mejor contexto.


Si sigues escribiendo largos prompts es porque necesitas organizar el contexto.


Dos pasos la primera vez, y luego iteras.

1. Estructura tu carpeta

CLAUDE COWORK/
├── CONTEXT/ ← quién eres, goals
├── REFERENCES/ ← docs externos
└── OUTPUTS/ ← trabajo terminado


→ CONTEXT/
3 archivos:
- about-me.md
- voice.md
- rules.md.


→ REFERENCES/
contratos marco, guias de marca, investigaciones, manuales internos.
Lo que informa tu trabajo pero no cambia por proyecto.


→ OUTPUTS/
el resultado final. Listo para exportar.


2. Configura Global Instructions

Settings → Cowork → Edit Global Instructions.

Puedes usar este prompt base:

Soy [Nombre], [Rol]. Antes de cualquier tarea, lee mis archivos en CONTEXT/. Muestra un plan breve antes de actuar. Pregunta si algo no esta claro. Si tu confianza en la respuesta es baja, dilo en lugar de inventar.



Con esto, hasta tu prompt mas lazy te entrega resultados calibrados. Claude ya sabe quien eres y lee los archivos relevantes antes de responder.


♻️ si esto te pareció útil, repostea. Alguien en tu red lo puede necesitar.
